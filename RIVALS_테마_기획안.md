# 작전노트 × RIVALS 테마 기획안

> 아드님 전용 게임화 레이어. 기존 작전노트 위에 "RIVALS 시즌을 플레이하는" 컨셉을 입힌다.
> 결과물 단계: **기획안 먼저** → 승인 후 코드.

---

## 1. 핵심 요약 (TL;DR)

- **컨셉**: 작전노트 = RIVALS 한 시즌. 아드님 = 플레이어. **과목별 숙제 = 무기**, **매일 미션 완수 = 승(Win)**, **숙제 미룸 = 패(Loss)**, **꾸준한 완수 = 무기 마스터리(스킬) 업그레이드**.
- **무기 매핑**: 아드님이 각 과제/교재에 RIVALS 실제 무기를 **직접 골라** 매핑한다(고정 추천 없음, 본인이 선택).
- **테마 시스템**: 기본은 일반 작전노트. 설정에서 **RIVALS 테마를 선택하면 그 페이지가 메인(홈)** 이 된다. 추후 다른 테마를 끼워 넣을 수 있는 **확장형 레지스트리 구조**로 설계.
- **데이터 전략**: 새 점수를 따로 쌓지 않고 **기존 학습 로그(dailyLogs/taskLogs)에서 전적·마스터리를 계산**한다(읽기 시 계산). 데이터 정합성 부담 0, 빠른 MVP.
- **동기부여 안전장치**: 아동 대상이므로 **무기 마스터리 XP는 절대 깎이지 않음**(누적만). 패배는 랭크/연승에만 영향. 패배 문구는 수치심 대신 "다음 매치 만회" 톤.

## 3. 게임 규칙 설계 (수치는 전부 튜닝 가능)

### 3-1. 승(Win) / 패(Loss) 정의

- **무기 단위**
  - 승: 그날 그 무기(과제)에 **예정된 분량을 완수** → +1 Win
  - 패: 예정된 날인데 **미완수로 그날이 지나감** → +1 Loss
- **매치(하루) 판정** — 그날 예정 무기들의 완수 비율로
  - 전부 완수 → **🏆 MVP 승리**
  - 일부 완수 → **판정승(부분 승)**
  - 0개 완수 → **패배**

### 3-2. 무기 마스터리 (과목별 숙련도 = "스킬 업그레이드")

> 핵심 원칙: **XP는 누적만, 절대 감소 없음.** 꾸준함이 곧 영구적 성장.

| 요소 | 값(예시) |
|---|---|
| 1승당 무기 XP | +10 |
| 연속 완수 보너스 | 연속 N일째 → +2×(N−1), 최대 +20 |
| 마스터리 레벨 임계값 | Lv1:0 / Lv2:30 / Lv3:80 / Lv4:160 / Lv5:280 / Lv6:450 / Lv7:700 |
| 레벨업 = | **"스킬 업그레이드"** 연출 + 스킨 티어 진화 |

**스킨 티어(레벨 구간 매핑)**

| 마스터리 레벨 | 스킨 티어 | 느낌 |
|---|---|---|
| Lv1–2 | 기본(Default) | 무광 |
| Lv3–4 | 실버 | 금속광 |
| Lv5–6 | 골드 | 화려함 |
| Lv7 (MAX) | 다이아 / 마스터 | 이펙트 |

### 3-2-1. 마스터리 천장 도달 후 — 프레스티지 / 이적 (확정)

무기가 최고 레벨(Lv7 MAX)에 닿아도 성장이 멈추지 않게, **도달 시 아드님이 둘 중 선택**:

- **A. 계속 키우기(프레스티지)**: 무기 그대로, 위에 **마스터 ☆** 누적(☆1→☆2→…). 골든 카모·이펙트 심화. 좋아하는 무기를 끝까지.
- **B. 새 무기로 이적**: 다 키운 무기는 **"마스터 무기"로 명예의 전당**에 보관(훈장), 그 과목에 새 무기를 장착해 Lv1부터. → 과목 단위인데도 시간이 지나며 무기가 수집됨.

> 천장은 빠르지 않다: Lv7 ≈ 누적 700 XP ≈ 약 70승 → 매일 해도 수개월. 닿을 때쯤 위 선택지가 새 동기로 이어짐.

| 천장 후 | 효과 | 성향 |
|---|---|---|
| A. 프레스티지 ☆ | 한 무기 깊이 | 무기 고집형 |
| B. 새 무기 이적 | 무기 수집·새 출발 | 다양성·수집형 |
| (자동) 둘 다 제공 | 그때그때 선택 | ✅ **확정** |

### 3-3. 플레이어 랭크 (계정 전체, ELO 기반)

- 시작 ELO **1000**
- 승(미션 완수) → **+20 ELO** / 패(미룸) → **−10 ELO**  *(승의 이득 > 패의 손실 = 아동 친화)*
- 연승 보너스: 3연승 +10, 5연승 +20, 7연승+ "킬스트릭" 추가 ELO

**랭크 티어 (실제 RIVALS 체계 차용)**

| 티어 | ELO 구간(예시) |
|---|---|
| 브론즈 I~III | 1000–1199 |
| 실버 I~III | 1200–1399 |
| 골드 I~III | 1400–1599 |
| 플래티넘 I~III | 1600–1799 |
| 다이아 I~III | 1800–1999 |
| 오닉스 I~III | 2000–2299 |
| 네메시스 | 2300–2599 |
| 아크네메시스 | 2600+ |

## 4. 무기 시스템 & 매핑 (아드님이 직접 선택)

RIVALS 실제 무기 53종 / 4카테고리 기반. 매핑 UI에서 카테고리별로 골라 과제에 붙인다.

| 카테고리 | 무기(예시) |
|---|---|
| **주무기(Primary)** | 어썰트 라이플 · 버스트 라이플 · 크로스보우 · 샷건 · 스나이퍼 · 보우 · RPG · 미니건 · 화염방사기 · 에너지 라이플 |
| **보조무기(Secondary)** | 핸드건 · 리볼버 · 우지 · 쇼티 · 스프레이 · 대거 · 플레어건 · 에너지 피스톨 · 엑소건 · 슬링샷 |
| **근접(Melee)** | 카타나 · 사이드(Scythe) · 주먹 · 나이프 · 배틀액스 · 체인소 · 라이엇 실드 · 트로웰 · 마울 |
| **유틸리티(Utility)** | 섬광/연막/수류탄류 등 |

- **매핑 단위: 과목당 무기 1개 (확정).** 같은 과목의 교재·과제는 전부 그 무기 XP로 합산 → 레벨업이 빠르고 무기고가 단순.
  - 예: `수학 = 어썰트 라이플`(수학 문제집·오답노트 합산), `영어 = 크로스보우`(영단어·영문법·영어인강 합산)
- **과제(tasks)의 과목 처리**: 현재 과제에는 `subject` 필드가 없음(교재만 있음). 매핑 화면에서 **각 과제를 과목에 배정**(기존 과목 선택 또는 신규)하면 과제 문서에 `subject`가 추가됨 → 관련 교재와 같은 무기로 합쳐짐. 미배정 과제는 `기타` 과목으로 묶임.
- 무기 카탈로그는 **코드 내 정적 config**(`WEAPONS`)로 관리 → 새 무기/아이콘 추가가 쉬움.
- 같은 무기를 여러 과목에 중복 매핑 허용(기본: 허용).

## 5. 테마 시스템 아키텍처 (확장형)

목표: **기본 테마가 디폴트, RIVALS 선택 시 그게 메인.** 나중에 테마를 계속 추가.

### 5-1. 테마 레지스트리 (코드 config)

```js
// app.js 또는 themes.js
const THEME_REGISTRY = {
  default: { id:'default', label:'기본',         home:'index.html',  nav:'default' },
  rivals:  { id:'rivals',  label:'RIVALS 작전',  home:'rivals.html', nav:'rivals'  },
  // 추후: space, rpg, pet ... 항목만 추가하면 끝
};
```

### 5-2. 설정 & 라우팅

- 설정 저장: `users/{uid}.settings.theme = 'default' | 'rivals'`
- **진입 라우팅**: 앱 첫 화면(또는 얇은 라우터)이 theme를 읽어 해당 `home`으로 보냄.
  - 깜빡임 방지를 위해 마지막 선택 테마를 `localStorage`에도 캐시 → 즉시 분기, Firestore는 동기화용.
- settings.html에 **테마 선택 카드**(레지스트리 자동 렌더) 추가.

### 5-3. 확장 규칙

새 테마 추가 = ① `THEME_REGISTRY`에 항목 1줄 + ② 그 테마의 home 페이지 1개. 게임 규칙(승/패/마스터리)은 공용 엔진으로 두고, 테마는 **표현(스킨)만** 바꾸는 구조 → 재사용 극대화.

## 6. 데이터 모델 (Firestore 확장 — 최소 변경)

```
users/{uid}
  settings.theme            : 'default' | 'rivals'              ← 신규
  weaponMap                 : { "수학":"assault_rifle",          ← 신규(과목→무기)
                                "영어":"crossbow", ... }
  weaponMeta (선택)         : { "수학": { prestige:0, retired:[] } } ← 천장/이적 기록
  gameProfile (선택, 캐시용) : { elo, rank, totalW, totalL, streak, updatedAt }

users/{uid}/tasks/{taskId}
  subject                   : "영어"                            ← 신규(매핑 화면에서 배정)
```

- **매핑은 과목 단위**: `weaponMap[subject] = weaponId`. 교재는 자기 `subject`로, 과제는 배정된 `subject`로 무기를 찾음. 새 교재/과제가 같은 과목이면 **자동으로 같은 무기 상속**.
- **전적/마스터리는 저장하지 않고 기존 로그에서 계산**(읽기 시 계산)이 기본.
  - 소스: `books/{id}/dailyLogs/{date}`(status: done/partial/extra), `taskLogs/{date_taskId}`(percent / checkedItems)
  - 승/패 판정: 예정일(교재 workDays / 과제 days)에 완료=승, 예정 과거일인데 진행 0=패, 부분진행=중립(아동 친화).
- 마스터리 XP는 **전체 기록 누적**(영구). 홈 성능이 더 필요하면 `gameProfile` 캐시는 Phase 2에서 추가.

## 8. 의사결정 비교표

### 결정 A — 전적/마스터리 계산 방식

| 방식 | 장점 | 단점 | 채택 |
|---|---|---|---|
| **읽기 시 계산**(로그에서 derive) | 정합성 100%, 마이그레이션 0, 기존 데이터 재사용 | 홈 로딩 시 계산량 | ✅ **MVP 채택** |
| 집계 카운터(완료 시 누적) | 홈 초고속 | 드리프트(어긋남) 위험, 쓰기 경로 추가 | 후순위 |
| 하이브리드(로그=진실 + 캐시) | 빠름+정확 | 구현 복잡 | 규모 커지면 |

### 결정 B — 패배 페널티 강도 (아동 동기부여)

| 강도 | 효과 | 위험 | 채택 |
|---|---|---|---|
| 강함(랭크·XP 둘 다 깎임) | 긴장감↑ | 좌절·이탈 위험 | ✗ |
| **약함(XP 보존, 랭크만 소폭↓)** | 꾸준함 보상, 회복 가능 | 긴장감 약간↓ | ✅ **채택** |

### 결정 C — 테마 라우팅

| 방식 | 장점 | 단점 | 채택 |
|---|---|---|---|
| **분리 페이지 + 진입 분기** | 기존 코드 안전, 확장 쉬움 | 페이지 추가 | ✅ **채택** |
| 한 페이지 조건부 렌더 | 파일 적음 | index.html 비대·리스크↑ | ✗ |

## 10. 동기부여 안전장치 (아동 설계 원칙)

- **성장은 영구적**: 무기 마스터리 XP는 깎이지 않는다. 꾸준함이 항상 남는다.
- **패배는 회복 가능**: 랭크는 다시 올릴 수 있고, "다음 매치에서 만회!" 톤. 수치심 문구 금지.
- **작은 승리 자주**: 과제 하나하나가 라운드 승 → 즉각 보상감.
- **비교는 자기 자신과**: 다른 사람 랭킹이 아니라 본인 시즌·연승과 비교.
- **연패 구제**: 며칠 빠져도 "리매치 찬스"로 복귀 동선 제공(완전 리셋 없음).
